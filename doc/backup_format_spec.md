# 装备录入工具备份文件格式规范 (Backup File Format Specification)

本文档详细描述了《烟雨江湖》装备录入工具导出的全量备份 JSON 文件的结构。理解此结构有助于手动编辑备份文件或进行数据排查。

## 1. 顶层结构
全量备份文件是一个 JSON 对象，包含以下两个主要属性：

```json
{
  "db": { ... },
  "settings": { ... }
}
```

-   `db`: 存储所有装备的录入序列数据。
-   `settings`: 存储工具的配置信息，主要是每个装备的“起始录入次数”。

## 2. `db` (数据库) 结构
`db` 是一个嵌套的对象，其结构反映了装备的分类、地点和具体装备的录入序列。

```json
{
  "装备类型": {                                // 如: "剑", "甲", "戒指", "项链", "护腕"
    "地点": {                                  // 如: "剑王阁", "铸剑山庄", "塞北", "明月峰"
      "装备名": [                             // 如: "巽风剑", "万仞重剑", "呼如木甲"
        {                                    // 录入的单个装备属性条目
          "q": "品质",                     // 字符串: "green" (绿), "blue" (蓝), "purple" (紫)
          "attrs": ["属性1", "属性2"],      // 数组: 基础属性列表，如 ["攻击", "命中"]
          "special": "特殊属性",             // 字符串: 特殊属性，如 "臂力+5"，如果没有则为空字符串
          "k": true                        // 布尔值: 是否被收藏 (keep)，true 或 false
        },
        null,                                // 表示该序列位置为空，未录入
        // ... 更多条目或 null ...
      ]
    }
  }
}
```

**示例 `db` 片段：**

```json
{
  "剑": {
    "剑王阁": {
      "巽风剑": [
        { "q": "green", "attrs": [], "special": "", "k": false },
        { "q": "blue", "attrs": ["速度", "气血"], "special": "臂力+5", "k": true }
        // ...
      ]
    }
  }
}
```

## 3. `settings` (设置) 结构
`settings` 是一个简单的键值对对象，每个键代表一个配置项，值是其对应的字符串形式。

目前主要包含“起始录入次数”的配置：

-   **键名格式**: `start_at_` + `地点` + `_` + `装备名`
    -   示例: `start_at_剑王阁_巽风剑`
-   **值**: 字符串形式的数字，表示从第几次开始录入 (1-based)。

**示例 `settings` 片段：**

```json
{
  "start_at_剑王阁_巽风剑": "1",
  "start_at_剑王阁_万仞重剑": "5",
  "start_at_塞北_呼如木甲": "16"
  // ...
}
```

## 4. 注意事项
-   **JSON 格式**: 备份文件必须是合法的 JSON 格式。手动修改时请确保语法正确。
-   **数据类型**: `q`、`special` 的值是字符串；`attrs` 是字符串数组；`k` 是布尔值；`start_at_` 的值是字符串数字。
-   **`null` 占位**: 序列数组中使用 `null` 来表示未录入的空位，请勿删除或修改其数据类型。
-   **兼容性**: 建议使用工具自带的备份/还原功能。手动修改可能导致数据损坏或还原失败。
